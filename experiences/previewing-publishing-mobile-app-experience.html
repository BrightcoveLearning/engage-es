Vista previa y publicación de una experiencia de aplicación móvil

En este tema, aprenderá cómo obtener una vista previa y publicar una experiencia de aplicación móvil.

La vista previa de una experiencia de aplicación móvil le permite ver cómo se mostrará la experiencia en un dispositivo móvil. Para obtener una vista previa de una experiencia de aplicación móvil, edite la experiencia y luego haga clic en Avance en el Editor de experiencias.

Hay dos opciones:

  • Vista previa adaptable: abre la experiencia en una nueva ventana del navegador para obtener una vista previa
  • Obtener enlace compartible: proporciona una URL de vista previa para que la vista previa se pueda compartir con otros usuarios.

La URL que se puede compartir que se genera es válida durante 72 horas.

Nota: Si ha configurado algún tiempo de espera de sesión como parte de cualquier configuración de control de acceso, el tiempo de espera de 72 horas anulará el tiempo de espera de sesión configurado. Cuando se publique la experiencia, se aplicará el tiempo de espera de la sesión (en su caso).

Publicar una experiencia de aplicación móvil

La publicación de una experiencia de aplicación móvil genera una clave de acceso. Brightcove Engage requiere la clave de acceso™ aplicación (iOS y Android) para acceder a la experiencia. Para obtener información sobre la instalación y el uso de la aplicación Brightcove Engage, consulte Uso de la aplicación Brightcove Engage.

Para publicar una experiencia de aplicación móvil, siga estos pasos:

Nota: Si recientemente has realizado cambios en los metadatos de vídeo o en las listas de reproducción, esos cambios pueden tardar unos minutos en aparecer en las experiencias publicadas.
  1. Haga clic en el nombre de la experiencia para abrirla en el Editor de experiencias.
  2. Haz clic en Publicar.
  3. En el cuadro de diálogo de confirmación de publicación, haga clic en Publicar ahora.
  4. Se publicará la experiencia y se mostrará la clave de acceso.

    Una vez que se publica la experiencia, la clave de acceso se mostrará como parte de los detalles de la experiencia en la página de inicio de Experiencias.

Uso de la aplicación Brightcove Engage

Una vez que se ha publicado una experiencia de aplicación móvil, se ingresa la clave de acceso en la aplicación móvil Brightcove Engage para ver la experiencia. Para obtener información sobre la instalación y el uso de la aplicación Brightcove Engage, consulte Uso de la aplicación Brightcove Engage.

Publicar cambios en una experiencia

Cuando realizas cambios en una experiencia, debes volver a publicarla. Las experiencias que deben volver a publicarse mostrarán el estado de la experiencia de los cambios no publicados.

Para volver a publicar los cambios, siga los pasos descritos en la Publicar una experiencia de aplicación móvil sección de este tema.

Cómo desconectar la experiencia de una aplicación móvil

Si una experiencia de aplicación móvil se desconecta, ya no será accesible desde aplicaciones móviles nativas. Para desconectar una experiencia de aplicación móvil, haga clic en una experiencia de aplicación móvil y luego haga clic en Desconectar. Para confirmar que la experiencia debe desconectarse, haga clic en Desconexión. Para que la experiencia vuelva a estar disponible, se debe volver a publicar la experiencia.

Nota: Si la experiencia fuera de línea no se elimina la experiencia.
