Vista previa y publicación de una experiencia web

En este tema, aprenderá a obtener una vista previa y publicar una experiencia web.

Las experiencias web se pueden previsualizar fácilmente durante el desarrollo. También se pueden compartir enlaces para obtener una vista previa de la experiencia. Cuando la experiencia esté completa, se puede publicar para que esté disponible públicamente.

Vista previa de una experiencia web

La vista previa de una experiencia web le permite ver cómo se mostrará la experiencia en dispositivos móviles, tabletas y computadoras de escritorio. Para obtener una vista previa de una experiencia web, edite la experiencia y luego haga clic en Avance en el Editor de experiencias.

Hay dos opciones:

  • Vista previa adaptable: abre la experiencia en una nueva ventana del navegador para obtener una vista previa
  • Obtener enlace compartible: proporciona una URL de vista previa para que la vista previa se pueda compartir con otros usuarios.

La URL que se puede compartir que se genera es válida durante 72 horas.

Nota: Si ha configurado algún tiempo de espera de sesión como parte de cualquier configuración de control de acceso, el tiempo de espera de 72 horas anulará el tiempo de espera de sesión configurado. Cuando se publique la experiencia, se aplicará el tiempo de espera de la sesión (en su caso).

Publicar una experiencia web

La publicación de una experiencia web la pone a disposición de los espectadores y genera una URL estática para el sitio. Para publicar una experiencia, siga estos pasos:

Nota: Si recientemente has realizado cambios en los metadatos de vídeo o en las listas de reproducción, esos cambios pueden tardar unos minutos en aparecer en las experiencias publicadas.
  1. Haga clic en el nombre de la experiencia para abrir el sitio en el Editor de experiencias.
  2. Haz clic en Publicar.
  3. En el cuadro de diálogo de confirmación de publicación, haga clic en Publicar ahora.
  4. Se publicará la experiencia y se mostrará la URL del sitio.

El bcvp0rtal dominio admite SSL de forma predeterminada, por lo que también se puede acceder a los sitios mediante https://, no se requiere configuración ni certificados SSL.

Una vez que se publica la experiencia, la URL se mostrará como parte de los detalles del sitio en la Proyectos página de inicio.

Publicar cambios en una experiencia web

Cuando realizas cambios en una experiencia, debes volver a publicarla. Las experiencias que deben volver a publicarse mostrarán el estado de la experiencia de los cambios no publicados.

Para volver a publicar los cambios, siga los pasos descritos en la Publicar una experiencia web sección de este tema.

Nota: Es posible que tenga que borrar la caché del navegador y las cookies si no aparecen cambios.

Tomando una experiencia web sin conexión

Si se desconecta una experiencia, ya no está disponible en su URL. Para llevar una experiencia sin conexión, haz clic en una fila y, a continuación, haz clic en Desconexión. Para confirmar que la experiencia debe desconectarse, haga clic en Desconexión. Para que la experiencia vuelva a estar disponible, se debe volver a publicar la experiencia. Una vez que la experiencia esté fuera de línea, ya no se podrá acceder a ella a través de su URL.

Nota: Si la experiencia fuera de línea no se elimina la experiencia.
