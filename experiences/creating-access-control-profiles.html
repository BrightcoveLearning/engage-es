Creación de perfiles de control de acceso

En este tema, aprenderá a crear perfiles de control de acceso.

De forma predeterminada, cuando Brightcove Engage™ se publica la experiencia, las experiencias son públicas:

  • Aplicaciones móviles - Cualquiera que tenga la clave de acceso y pueda acceder a la aplicación
  • Aplicaciones web - Cualquiera que tenga la URL del sitio podrá acceder al portal web.

Se pueden crear perfiles de control de acceso para que los usuarios se vean obligados a autenticarse con credenciales válidas antes de que puedan acceder a la aplicación / sitio. Brightcove Engage admite el estándar Security Assertion Markup Language 2.0 (SAML 2.0) para intercambiar datos de autenticación y autorización entre dominios de seguridad.

Brightcove Engage es compatible con la mayoría de los principales proveedores de identidad, incluidos:

  • Silbido
  • OneLogin
  • Auth0
  • ADFS 2.0/3.0
  • Google G Suite
  • Santo y seña
  • Okta
  • Salesforce.com

También funcionarán otros proveedores que utilicen el estándar SAML 2.0.

Nota: Brightcove Engage requiere SAML 2.0 para la autenticación de inicio de sesión único. SAML 1.1 no se admite.

Al configurar su software SSO, asigne los siguientes atributos:

  • firstName se asigna a First Name
  • lastName se asigna a Last Name
  • enviar mapas por correo electrónico al correo electrónico

Creando un nuevo perfil

Los perfiles de control de acceso se crean como parte de la configuración del proyecto y luego se asignan a las experiencias.

Para crear un perfil de control de acceso, siga estos pasos:

  1. Inicie sesión en Brightcove Engage.
  2. Desde el Proyectos página, haga clic en Ajustes.
  3. En la navegación izquierda, haz clic en Perfiles de control de acceso. Aparecerá la página Perfiles de control de acceso con una lista de perfiles (si los hay) que se han creado.
  4. Haz clic en Crear perfil.
  5. Asigne un nombre al perfil.
  6. (Opcional) Configurar un Hora de término de la sesión. Esta es la cantidad de tiempo que un usuario puede iniciar sesión en una aplicación antes de que deba volver a autenticarse. Algunas notas:
    • Su proveedor de SSO debe estar configurado para forzar la reautenticación o tener un tiempo de espera de sesión menor que este tiempo de espera. Si no es así, lo redireccionaremos a su proveedor de SSO después de nuestro tiempo de espera e inmediatamente emitirán una respuesta de que el usuario ha iniciado sesión.
  7. Selecciona el SSO opción. Aparecerán las opciones de configuración de SSO.
  8. Si tiene un archivo de exportación de sus metadatos SAML 2.0, haga clic en Seleccione Archivo y luego busque y seleccione el archivo de metadatos. Esto rellenará previamente el Punto final SAML 2.0 y el Certificado X.509 los campos.
  9. Introduzca un valor para el endpoint SAML 2.0 (HTTP). Este valor debe estar disponible en su proveedor de SAML.
  10. Ingrese un valor para Certificado X.509. Este valor debe estar disponible en su proveedor de SAML.
  11. (Opcional) Ingrese el Reemplazo de ID de entidad / emisor de SAML. Ingrese un valor aquí para anular opcionalmente el emisor SAML.
  12. (Opcional) Ingrese el Anulación de URL ACS. Ingrese un valor aquí para anular opcionalmente la URL de ACS.
  13. (Opcional) Ingrese el Reemplazo de AuthnContext de SAML. Ingrese un valor aquí para anular opcionalmente SAML AuthnContext.
  14. (Opcional) Ingrese los atributos de SAML que desee que se pasen a su JavaScript externo.
  15. (Opcional) Verificar Exponer todos los atributos para exponer todos los atributos de SAML para su uso en JavaScript externo.
  16. Haz clic en Guardar.
  17. Verifique que el perfil que creó aparece en la lista de Perfiles de control de acceso.

Para editar o eliminar un perfil de control de acceso, haga clic en editar () o suprimir () icono asociado al perfil. Una vez que se han creado los perfiles de control de acceso, se pueden asignar a un sitio como parte de las propiedades del sitio. Para obtener información, consulte Controlar el acceso a una experiencia.

Configuración de la URL de ACS

Su proveedor de SSO requerirá una URL de servicio del consumidor de afirmación (URL ACS). La URL de ACS aparece en la sección SSO de la tabla Perfiles de control de acceso.

La URL de ACS tiene como valor predeterminado https://auth.bcvp0rtal.com/login/callback. También puede anular la URL ACS proporcionando su propia URL. Ambas opciones se pueden utilizar para configurar proveedores de SSO más estrictos.

Nota: Cuando utilice un flujo de trabajo de SSO iniciado por IDP, la URL de ACS especificada en su software de SSO debe apuntar a un sitio de Brightcove Engage que esté configurado con ese perfil de control de acceso. En este caso, no se debe utilizar la https://auth.bcvp0rtal.com/login/callback URL.
